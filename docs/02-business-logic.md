# Business Logic & The Data Flywheel

The codebase is engineered to execute AIgenda's "Two-Layer Model" and the "Data Flywheel." This document explains how the current logic achieves the Core Vision.

## 1. Layer 1 (Workspace) & Layer 2 (AI Intelligence) Interaction
- **Layer 1 (The Entry Point)** is actively represented by the `Services` namespace (`TaskService`, `NoteService`, `FocusSessionService`, etc.). It captures how a user naturally works without forcing them into foreign paradigms.
- **Layer 2 (The Differentiator)** relies entirely on the data footprint left by Layer 1. Currently, Layer 1 generates the base schemas (Tasks completed vs. overdue, Time spent in Focus Sessions, Journal entries in Notes) that Layer 2 will soon ingest to produce personalized coaching and behavioral insights.

## 2. How the Integrations Logic Serves Layer 1
To solve the "Fragmentation Problem," the system leverages `GitHubIntegrationService`, `GmailIntegrationService`, and `GoogleCalendarIntegrationService`.
- **Purpose**: Instead of making users switch tabs, these services poll and ingest data (commits, PRs, meetings, emails) into the AIgenda database.
- **Business Value**: Centralizing this data allows Layer 2 to evaluate patterns holistically. Without integrations, the AI's view of the user's workload would be incomplete (e.g., missing GitHub coding time or Google Calendar meetings).

## 3. The Data Flywheel Implementation
The current services lay the exact foundation required for the step-by-step logic of the Flywheel:
1. **User Does Work**: Managed via `TasksController`, `FocusSessionController`, and `NotesController`.
2. **Data Is Generated**: Standardized in the database using comprehensive schemas (e.g., tracking focus session durations, task creation vs completion timing).
3. **AI Analyzes (Next Evolution)**: Data generated by `FocusSessionService` and `TaskTracking` will be fed to the Intelligence Engine.
4. **Insights Delivered**: Will eventually be exposed via a new `InsightsService` or Dashboard logic.
5. **User Acts & Grows**: As the user corrects behavior, new `Tasks` and `FocusSessions` enter the system, closing the loop.
